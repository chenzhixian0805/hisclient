# hisclient

## 项目概述

hisclient 是一套医院信息管理系统（HIS）的前端客户端，采用 Vue 3 与 Element Plus 技术栈构建。系统面向医院门诊、医生工作站及药房三大业务场景，提供挂号、诊疗、病历、药品等全流程的信息化支持，帮助医疗机构实现日常业务的高效运转与数据的规范化管理。

## 技术栈

- **Vue 3**：前端框架，采用 Composition API 进行组件逻辑组织
- **Vue Router 4**：官方路由管理器，实现单页应用的多模块页面跳转与嵌套路由控制
- **Element Plus**：基于 Vue 3 的 UI 组件库，提供表格、表单、对话框等医疗系统常用组件
- **Element Plus Icons**：图标库，全局注册后供各业务模块统一调用
- **Vue CLI 5**：项目脚手架与构建工具

## 技术架构

项目采用典型的单页应用架构，入口文件通过 `createApp` 创建 Vue 应用实例，并挂载 Vue Router 与 Element Plus。整体使用 Vue 3 的 Composition API 进行组件逻辑组织，UI 层面选用 Element Plus 组件库以保证界面的一致性与开发效率。

路由层面采用 `createWebHashHistory` 模式，以 `/home` 为主框架页面，各业务功能以嵌套路由形式挂载其下。医疗记录与开药两个模块还设计了二级子路由，分别承载疾病添加与药品添加的独立页面。所有业务视图均通过路由懒加载（`import()`）方式引入，以控制首屏资源体积。

## 核心业务模块

### 门诊挂号

门诊挂号模块支撑患者到院后的现场挂号流程，提供挂号信息的录入与挂号记录的查询回溯功能。通过该模块，挂号处工作人员可快速完成患者分诊与号源分配。

### 病历管理

电子病历模块用于患者诊疗过程中病历资料的记录与调阅，将传统纸质病历转化为结构化电子数据。模块内还嵌套了疾病管理子页面，方便医生在书写病历时直接维护疾病信息库。

### 科室诊疗

科室诊疗模块覆盖内科与外科两大核心科室的日常工作流程。医生可在工作站内完成患者接诊、病情记录，并通过在线开药功能直接生成处方，实现诊疗与用药环节的无缝衔接。开药模块下还嵌套了药品添加子页面，支持医生在处方过程中快速补充新药信息。

### 药品管理

药品管理模块面向药房场景，提供药品库存的实时监控、药品发放记录的管理，以及药品基础信息的维护功能，确保药房对药品流转的全流程可控、可查。

### 患者信息

患者信息模块集中管理患者的基础档案资料，支持信息的录入、修改与查看。个人中心则为不同角色的系统使用者提供个人信息的管理入口。

## 系统特点

- **模块化路由设计**：以首页为布局框架，各业务功能按模块拆分为独立子路由，支持医疗记录、开药等复杂场景的二级页面嵌套
- **路由懒加载**：业务视图按需加载，降低首屏渲染时间，提升系统打开速度
- **响应式数据流**：基于 Vue 3 的响应式系统，保证界面状态与业务数据实时同步
- **统一 UI 风格**：全系统采用 Element Plus 组件与图标库，交互方式统一，降低医护人员学习成本
- **完整业务链路**：覆盖门诊挂号、科室诊疗、病历书写、在线开药到药房发药的全流程，实现院内信息的闭环流转
